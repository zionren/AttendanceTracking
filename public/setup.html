Follow these steps to set up your Supabase database:

  1. Go to the Supabase dashboard
  2. Create a new project if you haven't already
  3. Once in the project page, click the "Connect" button on the top toolbar
  4. Copy URI value under "Connection string" ‚Üí "Transaction pooler"
  5. Replace [YOUR-PASSWORD] with the database password you set for the project
Important: Keep your database credentials secure and never commit them to version control.

‚öôÔ∏è Environment Configuration

Create a .env file in your project root directory with the following variables:

Required Environment Variables:
# Database Configuration DATABASE_URL=postgresql://[username]:[password]@[host]:[port]/[database] SUPABASE_ANON=your_supabase_anon_key SUPABASE_KEY=your_supabase_service_role_key # Server Configuration (Optional) PORT=8000

How to Get Supabase Keys

  1. In your Supabase project dashboard, go to "Settings" ‚Üí "API"
  2. Copy the anon public key for SUPABASE_ANON
  3. Copy the service_role key for SUPABASE_KEY

üîß Troubleshooting

Common Issues

  • Database Connection Error: Verify your DATABASE_URL is correct and the database is accessible
  • Port Already in Use: Change the PORT in your .env file or stop other services using port 8000
  • Missing Dependencies: Run npm install to install all required packages
  • Supabase Errors: Check your SUPABASE_ANON and SUPABASE_KEY are correctly set

Database Initialization

The application automatically creates the required tables and default mains on startup. If you encounter issues:

  1. Check the server console for initialization messages
  2. Verify your database permissions allow table creation
  3. Ensure the database specified in DATABASE_URL exists
